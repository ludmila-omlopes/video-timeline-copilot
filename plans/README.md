## Findings audited but not planned (recorded so they aren't re-audited)

- **Transcript cache keyed by file stem** (`helpers/transcribe.py:107`, consumed stem-based by `validate_edl.py:11`, `draft_silence_cut.py:96`, `export_srt.py:30`): two sources with the same filename in different folders collide in `edit/transcripts/`; an existing transcript is served even if the source changed (no mtime/hash check). Real but M-effort (cache key migration touches 4 modules + docs); plan it when multi-source workspaces become common.
- **FCPXML asset duration computed from the first timeline only** (`helpers/export_fcpxml.py:92-109`): in multi-timeline EDLs, a source's asset `duration` is the max `source_end` used in the timeline where it *first* appears; a later timeline using later footage exceeds the declared asset duration. Fix is small (use `media_index.json` duration, fall back to global max across timelines); deferred as lower-frequency than the planned items.
- **Validator doesn't flag overlapping record ranges** on the same track (`helpers/validate_edl.py`): overlaps silently produce overlapping FCPXML spine clips and a too-long preview. Worth a warning in `cut_quality_warnings`; deferred.
- **`build_resolve_project.py:160` writes float fps into Resolve settings** (`"29.97002997..."` instead of `"29.97"`): same family as plan 002, but needs a machine with Resolve Studio to verify; noted in plan 002's maintenance notes.
- **`install.md` command list is stale** (missing `draft-silence-cut`, `update-fcpxml`, `render-preview`, `qa-preview` in the `vtc <cmd> --help` list): cosmetic; fold into the next docs pass.
